## 2024-05-18 - Fix DOM-based XSS via URL Search Parameters
**Vulnerability:** User-controlled parameters (`?join=roomName`) accessed via `URLSearchParams` were interpolated directly into raw HTML template strings and injected into the DOM via `.innerHTML` in multiple locations (`showAutoJoinOverlay` and `showNamePromptModal`).
**Learning:** In a vanilla JS extension environment, taking URL parameters directly to `.innerHTML` creates a very straightforward DOM-based Cross-Site Scripting (XSS) vulnerability. An attacker can craft a malicious link pointing to the extension's page (e.g., via background/content script triggers) and execute arbitrary JS code in the context of the extension.
**Prevention:** Avoid interpolating variables, especially URL parameters or other external input, into HTML template strings assigned to `.innerHTML`. Always use safe DOM APIs like `document.createElement`, or use empty container elements within `.innerHTML` and populate them safely afterward using `.textContent` or `.innerText` like: `element.querySelector('#id').textContent = userInput;`.
