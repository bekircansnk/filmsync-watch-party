## 2024-07-21 - Custom UI Components functioning as buttons
**Learning:** In the FilmSync extension, the platform cards (`.platform-card`) function as buttons (triggering a URL creation) but were originally built as `div` elements relying only on `click` event listeners without native semantic tags or keyboard event listeners. Consequently, users navigating with a keyboard could not properly tab to or activate the buttons.
**Action:** When implementing custom DOM elements (like `div`s) functioning as buttons rather than native `<button>` or `<a>` elements, always include `role="button"`, `tabindex="0"`, descriptive `aria-label`s, `:focus-visible` styling, and a `keydown` listener to handle `Enter` and `Space` key actions explicitly to maintain accessibility. Ensure decorative elements like emojis inside these are hidden from screen readers using `aria-hidden="true"`.
